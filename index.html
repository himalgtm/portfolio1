Kozi.REST - AI-Powered Sleep Tracker

View Devpost

Kozi.REST is an AI-powered bedside sleep companion designed to help users understand and improve their sleep quality through environmental and behavioral data. I led the integration of Raspberry Pi sensors that monitored ambient temperature, humidity, sound levels, and body motion throughout the night. These readings were processed locally and summarized every morning through an AI-generated audio briefing created with Gemini API and ElevenLabs TTS. The system included a lightweight web dashboard that visualized sleep trends, identified anomalies, and delivered insights into how room conditions affect rest quality. Beyond just logging data, Kozi.REST personalized feedback using an adaptive AI model that adjusted its recommendations over time, bridging hardware sensing with ambient intelligence in a compact IoT setup.

RunSafe - Smart Running Tracker

View Devpost

RunSafe was built to make outdoor running safer by merging geospatial analytics with environmental health data. The app retrieves live AQI (air-quality index) data and computes the ‚Äúhealthiest‚Äù route among multiple alternatives provided by Google Maps. I developed the backend pipeline that decodes route polylines, samples coordinate points, synchronizes them with hourly AQI data from Open-Meteo, and computes cumulative exposure scores for each route. The client visualizes these scores as color-graded paths, giving runners real-time comparisons. RunSafe also allows users to log daily symptoms (like coughing or fatigue) and auto-generates weekly health reports in PDF, formatted for physician review. The project was inspired by the Rice IDSO Challenge on Air Quality & Lung Health and aims to connect environmental data with individual health outcomes.

Backlog Buddy - Cloudflare AI Backlog Agent

Live Demo GitHub

Backlog Buddy is a small AI agent hosted on Cloudflare that helps a developer triage and act on their issue backlog. You chat with it in plain English, and it replies with a concrete plan: which issue to work on, what git branch to create, and which commands to run next.

Under the hood, a BacklogAgent Durable Object stores a seeded list of issues plus recent notes. Each message is appended to that state, then the agent builds a prompt with the backlog + conversation and calls @cf/meta/llama-3.3-70b-instruct-fp8-fast via Workers AI. Responses are streamed back through the Worker to the chat UI.

Responsibilities

  • Designed the agent‚Äôs state model (issues, notes, lastUpdated) on top of Durable Objects.
  • Implemented the chat endpoint and session routing so each URL session maps to its own agent.
  • Built a minimal but responsive HTML chat UI served directly from the Worker.

MyGarage - A Simple Car Tracker

GitHub

MyGarage is a clean and minimal web app I built to keep track of the cars. It lets me add details like the make, model, year, mileage, fuel type, VIN number, and more. If anything changes like mileage or service date; I can update it. And when I no longer need a car in the list, I can easily delete it. What makes it more fun (and useful) is the Scan VIN feature. Instead of typing out the full 17-character VIN, I can just click a button, open my camera, and it reads the VIN directly from the image using Google Vision OCR. It‚Äôs quick, accurate, and saves time. The app is built using Node.js, Express, and MongoDB (via MongoDB Compass) running locally to store all the car data. It also uses WebSockets to sync changes across tabs or devices in real time so if I update something on my phone, it instantly updates on my laptop too. The frontend is built with vanilla JavaScript and Tailwind CSS for a simple but sharp retro look.
