SAM 2 Installation & Overview ‚Äî Windows Setup + Geospatial Integration

Getting Meta‚Äôs Segment Anything Model 2.1 (SAM 2.1) running on Windows comes down to a clean Python env, the right PyTorch ‚áÑ CUDA combo, and the model checkpoints. Here‚Äôs the streamlined path I use on Windows.

1) Environment & PyTorch (CUDA)

# create & activate (venv example; conda works too)
python -m venv .venv
.\.venv\Scripts\activate

# install PyTorch matching your CUDA (example: CUDA 12.4)
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu124

# core deps
pip install opencv-python matplotlib notebook rasterio shapely
# verify CUDA availability & versions
python -c "import torch; print(torch.cuda.is_available(), torch.version.cuda)"

2) Clone & Install SAM2

git clone https://github.com/facebookresearch/segment-anything-2.git
cd segment-anything-2
pip install -e .

If the repo ships a download_ckpts.sh script, run it under Git Bash/WSL or download checkpoints manually from the model card and place them in a known folder (e.g., ./checkpoints/).

3) Quick GPU Smoke Test

import torch, cv2
from segment_anything_2 import build_sam2

device = "cuda" if torch.cuda.is_available() else "cpu"
model = build_sam2(checkpoint="checkpoints/sam2_hiera_b+.pt").to(device)

img = cv2.imread("your_image.jpg")[:,:,::-1]  # BGR‚ÜíRGB
# minimal point prompt example (x=100,y=150)
point = [[100,150]]; label = [1]
mask = model.segment(img, point_coords=point, point_labels=label)
print("Mask shape:", mask.shape)

4) Troubleshooting Notes

  • Match PyTorch wheel to your installed CUDA (and VS 2022 C++ build tools for any native extensions).
  • If pip install -e .[demo] is flaky on Windows, install listed demo deps individually (OpenCV, Matplotlib, Jupyter).
  • If CUDA isn‚Äôt detected, verify your PATHs and torch.cuda.is_available().

How SAM 2.1 Works (Quick Tour)

  • Image Encoder (Hiera ViT): streams frames/images into multiscale embeddings.
  • Prompt Encoder: points/boxes (sparse) and masks (dense) guide what to segment.
  • Memory Bank + Attention: for videos, past-frame features guide current masks.
  • Mask Decoder: transformer head outputs masks (and ‚Äúobject absent‚Äù when needed).
  • Memory Encoder: writes current-frame results back to memory for future frames.

5) Geospatial: Segment-Geospatial (SAM on GeoTIFF)

Use segment-geospatial to run SAM/SAM2 on NAIP/CHM or other GeoTIFFs and export vectors for GIS. 

pip install segment-geospatial
# Automatic mask generation (whole image)
from samgeo import SamGeo2
sam = SamGeo2(automatic=True, model_id="sam2-hiera-large")
sam.generate(image="sample_naip.tif", output="output_mask.tif")

Large scenes can yield hundreds of segments. If you hit uint8 save errors (IDs > 255), write as uint16 GeoTIFF or vectorize directly. 

# Prompted prediction (click-like point)
sam = SamGeo2(automatic=False, model_id="sam2-hiera-large")
sam.set_image("sample_naip.tif")
mask = sam.predict(point_coords=[(x, y)], point_labels=[1])

RunSafe - Smart Running Tracker

View Devpost

RunSafe was built to make outdoor running safer by merging geospatial analytics with environmental health data. The app retrieves live AQI (air-quality index) data and computes the ‚Äúhealthiest‚Äù route among multiple alternatives provided by Google Maps. I developed the backend pipeline that decodes route polylines, samples coordinate points, synchronizes them with hourly AQI data from Open-Meteo, and computes cumulative exposure scores for each route. The client visualizes these scores as color-graded paths, giving runners real-time comparisons. RunSafe also allows users to log daily symptoms (like coughing or fatigue) and auto-generates weekly health reports in PDF, formatted for physician review. The project was inspired by the Rice IDSO Challenge on Air Quality & Lung Health and aims to connect environmental data with individual health outcomes.
